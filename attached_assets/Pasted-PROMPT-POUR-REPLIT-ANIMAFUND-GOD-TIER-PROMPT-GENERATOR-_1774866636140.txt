PROMPT POUR REPLIT – ANIMAFUND GOD TIER PROMPT GENERATOR
CONTEXTE GÉNÉRAL
Tu vas concevoir un générateur de prompts God Tier pour une plateforme appelée ANIMAFUND. Cette plateforme ne produit pas directement les documents finaux (dossiers de financement, pitch decks, art bibles, etc.). Elle produit des prompts parfaitement structurés destinés à être injectés dans des IA génératives externes comme Meta.ai, Roboneo, Google Flow Studio, Midjourney, Runway, GPT-5.2, Claude, etc.

L’objectif est que n’importe quel utilisateur (studio d’animation, développeur jeu vidéo, producteur) puisse, via une interface simple, sélectionner un module, remplir des formulaires, et obtenir 3 variantes de prompts God Tier (expert, créatif, template) pour chaque sous-module, avec un système de scoring garantissant une qualité irréprochable.

Le résultat final doit être un outil fonctionnel sur Replit, avec une UI cartographique, des formulaires dynamiques, et un moteur de prompts basé sur GPT-5.2 ou Claude.

PRINCIPES FONDAMENTAUX
Zéro production directe : L’outil ne génère jamais de PDF, PPT, ou document final. Il ne génère que des prompts.

Qualité absolue : Chaque prompt doit être "God Tier" – c’est-à-dire si parfaitement structuré qu’injecté dans Meta.ai ou autre, le résultat est indistinguable d’un travail de consultant expert.

Modularité : 10 modules principaux, chacun avec ses sous-modules. Chaque sous-module peut être généré indépendamment.

Scoring obligatoire : Un système de scoring (0-100) évalue chaque prompt. En dessous d’un seuil (ex: 85), le prompt est automatiquement régénéré avec des instructions d’amélioration ciblées.

3 variantes par prompt : Chaque sous-module produit 3 prompts de styles différents : Expert/Formel, Créatif/Incisif, Structuré/Template.

Formulaires intelligents : Un formulaire commun (projet) + un formulaire spécifique par module. Les données du formulaire commun alimentent tous les prompts pour garantir la cohérence.

Interface cartographique : Chaque module est une carte colorée avec ses sous-modules affichés, un statut (généré ou non), et un score visible.

7. Système de scoring (prompts/scoring.py)
Le scoring est effectué par le LLM lui-même. Chaque prompt généré est noté sur 100 selon les critères suivants (à envoyer au LLM avec la consigne "Note ce prompt de 0 à 100 selon ces critères") :

Critère	Pondération	Description
Complétude	20%	Toutes les sections du template sont présentes et remplies
Spécificité	20%	Le prompt contient des éléments propres au projet, pas du générique
Précision réglementaire	15%	Référence aux bonnes normes avec liens/versions
Clarté structure	15%	La structure de sortie est explicite et non ambiguë
Few-shot quality	10%	Les exemples sont pertinents et bien choisis
Contrôle qualité	10%	Instructions explicites pour éviter hallucinations
Adaptation plateforme	10%	Le prompt est optimisé pour la plateforme cible
Seuils :

Score < 70 : régénération automatique avec feedback

70 ≤ score < 85 : livraison avec mention "bon"

85 ≤ score < 95 : livraison avec mention "excellent"

Score ≥ 95 : livraison avec mention "God Tier"

Régénération :
Si score < SEUIL_GOD_TIER et itération < MAX_ITERATIONS :

Extraire les critères faibles du retour du LLM

Générer un nouveau prompt avec instruction : "Améliore spécifiquement les points suivants : [critères faibles]"

Re-scorer, jusqu’à 3 itérations.

8. Interface utilisateur (Streamlit)
Page principale :

Bandeau supérieur : Titre "ANIMAFUND – God Tier Prompt Generator"

Formulaire commun (dépliable, avec état sauvegardé en session)

Grille de cartes : 10 cartes colorées, chacune avec :

Nom du module

Icône

Liste des sous-modules (avec checkbox ou statut)

Score global du module (moyenne des sous-modules générés)

Bouton "Générer tout le module" ou "Générer ce sous-module"

Zone de résultats :

Quand un sous-module est généré, afficher :

3 onglets (Expert, Créatif, Template)

Score et nombre d’itérations

Boutons de copie et de téléchargement (fichier .md)

Barre latérale :

État de progression (nombre de sous-modules générés / total)

Sélection du seuil de scoring (paramètre ajustable)

Sélection de la plateforme cible par défaut (Meta.ai, Roboneo, etc.)

Comportement attendu :

Les données du formulaire commun sont sauvegardées dans st.session_state pour persister entre les pages.

La génération d’un sous-module appelle les fonctions du module correspondant, injecte les données des formulaires, lance le scoring, et sauvegarde le résultat dans un fichier JSON + Markdown dans prompts/generated/.

L’utilisateur peut régénérer un sous-module (nouvelle itération) ou passer au suivant.

9. Backend LLM (llm/client.py)
Wrapper unifié pour appeler GPT-5.2 ou Claude.

Fonctions nécessaires :

call_llm(prompt_system, prompt_user, model="gpt-5.2", temperature=0.7) → retourne le texte généré.

score_prompt(prompt_text) → retourne un dictionnaire {score_total, criteres_details, weak_points}.

generate_variants(prompt_base, style) → retourne une version du prompt adaptée au style (expert, creatif, template).

Les appels doivent gérer les retries, les timeouts, et les quotas.

10. Sauvegarde et chargement
Chaque prompt généré est sauvegardé dans prompts/generated/[module_id]/[sous_module_id]/ avec :

prompt_v1_expert.md

prompt_v2_creatif.md

prompt_v3_template.md

metadata.json contenant : score, iterations, date, version, plateforme_cible, variantes utilisées.

Un fichier progress.json global garde la trace de ce qui a été généré pour la session en cours (ou pour l’utilisateur si authentification ajoutée plus tard).
